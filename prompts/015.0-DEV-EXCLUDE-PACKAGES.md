# 015.0-DEV-EXCLUDE-PACKAGES: Package Exclusion via Config File

## Release Goal

**Release 0.9: Package Exclusion**

Allow specific packages to be excluded from dry-aged-deps analysis via the config file, with a mandatory reason for each exclusion.

## How This Story Contributes

Some packages cannot be upgraded due to ecosystem constraints (e.g., a plugin not yet compatible with a new major version). This feature lets teams acknowledge those packages and exclude them from output, reducing noise while maintaining an audit trail of why each exclusion exists.

## User Story

**Format**: So that known-blocked packages don't clutter my outdated report, as a developer, I want to exclude specific packages from analysis via the config file with a mandatory reason.

**INVEST Criteria Compliance**:

- **Independent**: Standalone feature, config-file only
- **Negotiable**: Exclusion format and output details can be refined
- **Valuable**: Reduces noise from packages that cannot be upgraded
- **Estimable**: Clear scope - config validation + filtering + output
- **Small**: Single iteration
- **Testable**: Can verify filtering behavior and output

## Acceptance Criteria

- [ ] **Config Format**: `exclude` key in `.dry-aged-deps.json` maps package names to reason strings
- [ ] **Reason Required**: Empty string reasons are rejected with exit code 2
- [ ] **Validation**: Non-object values and non-string reasons are rejected with exit code 2
- [ ] **Filtering**: Excluded packages are removed before age/security analysis
- [ ] **JSON Output**: Excluded packages appear in a separate `excluded` array with name and reason
- [ ] **XML Output**: Excluded packages appear in a separate `<excluded>` section
- [ ] **Table Output**: A note is printed showing how many packages were excluded

## Requirements

- **REQ-EXCLUDE-CONFIG**: Support `exclude` object in config file (package name -> reason string)
- **REQ-EXCLUDE-VALIDATION**: Validate exclude is an object with non-empty string values
- **REQ-EXCLUDE-FILTER**: Remove excluded packages before building rows and applying filters
- **REQ-EXCLUDE-OUTPUT**: Include excluded package info in JSON/XML output; show count note in table output

## Config Format

```json
{
  "exclude": {
    "eslint": "Blocked by eslint-plugin-traceability incompatibility with ESLint 10 API",
    "@eslint/js": "Must upgrade together with eslint"
  }
}
```

## Dependencies

- **010.0-DEV-CONFIG-FILE-SUPPORT**: Requires config file infrastructure
